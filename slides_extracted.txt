===== PAGE 9 =====
AGENT WORKFLOW
Quality Signalling & the Reflection Loop
Every retrieval reports quality = GOOD / WEAK.
Based on average vector relevance vs a 0.30 threshold.
Calibrated to the MiniLM cosine range — not arbitrary.
The LLM uses this signal to reformulate, escalate, or stop.
Reflector (separate node) scores completeness + grounding after
generation.
Fail + budget left + not web-grounded → loop back and re-
retrieve.
A genuine closed loop: generator → reflector → retriever →
generator.
===== PAGE 10 =====
CHALLENGE → SOLUTION
Web-Guard: A Real Bug I Found and Fixed
Bug: a web-grounded answer that fails reflection would re-retrieve documents.
→ discards the live web result, falls back to stale general knowledge. Strictly worse.
Fix: _should_continue returns "end" on failed reflection IF answer is web-grounded.
The web answer is preserved instead of being thrown away.
Locked in by trajectory test T5 — fails immediately if the guard is removed.
A bug, its fix, and a regression test = the engineering maturity I want to show.

RETRIEVAL DESIGN
Why Hybrid + RRF + Reranker
Hybrid covers complementary failure modes:
BM25 nails exact technical terms and acronyms — FD004, OSM-
PINN.
Dense vectors handle paraphrase and semantic similarity.
My corpus is acronym-heavy research papers — I genuinely need
both.
RRF (rank-based) — no per-corpus weight tuning; immune to
incomparable score scales.
CrossEncoder — scores query + chunk jointly; the single biggest
precision lever.
vector_score preserved — reranker logit is for ordering only.

===== PAGE 24 =====
Limitations & Future Improvements
Embedding ceiling — MiniLM 384-d is lightweight.
Upgrading to a bge / mxbai-class 768–1024-d model + re-indexing is the highest-
leverage win.
No query decomposition — multi-hop handled by in-loop reformulation, not
sub-query planning. Naming limitations
unprompted
builds credibility.
No persistent trace store — traces stream live but aren't queryable post-hoc via
an API.
Engineering integrity
Latency — P95 ≈ 22–26 s on multi-tool paths.
Fine for a streamed async UX; not a low-latency synchronous API yet.
